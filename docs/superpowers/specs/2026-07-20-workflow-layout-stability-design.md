# 工作流布局清晰度与拖拽稳定性设计

## 背景

当前工作流设计器存在三个相互叠加的问题：

1. 套用模板后，数据库为了保留历史业务引用，会将旧状态标记为禁用并创建新的模板状态。设计器仍绘制这些禁用状态，因此出现同名重复节点。
2. 同状态操作（例如编辑、补充信息、转派、查看历史）被绘制成自环边。默认需求模板包含大量同状态操作，自环会占用主流程周围的路由空间，迫使其他边向画布外围绕行。
3. 拖拽节点时，设计器把全部边切换成简化预览，并临时移除边界；松手后又恢复完整路由、重新计算画布并自动限制视口，造成连线和页面位置跳变。

## 目标

- 流程图优先表达状态之间的变化，接近“两行主流程 + 独立回退通道”的阅读结构。
- 所有流转按钮仍可见、可选择、可配置，不改变工作流运行语义。
- 同状态操作不再绘制自环线，而是作为节点下方的紧凑操作按钮展示。
- 停用状态始终可见、可重新启用，但与启用主流程明确分区。
- 拖拽节点时，视口和无关连线保持稳定。
- 清理模板套用产生的禁用重复状态，并防止后续重复套用再次产生重复状态。

## 非目标

- 不改变工作流运行时如何计算可用按钮。
- 不改变权限、处理人、校验器、条件、自动化或表单配置的业务语义。
- 不自动保存整理或拖拽后的坐标。
- 不删除没有同名启用状态可承接引用的禁用状态。
- 不引入新的流程图库或自动布局依赖。

## 画布表达

### 状态节点

- 启用状态使用现有实色节点样式并参与主流程布局。
- 停用状态使用白底、灰色虚线边框和“已停用”标记。
- 停用状态排列在主流程下方的独立区域，不参与启用主流程分层。
- 停用状态仍可选中，并可在抽屉中重新启用。

### 状态变化流转

当 `from_state_id !== to_state_id` 时，流转继续显示为正交连线和可点击的按钮标签。

- 主流程从左到右。
- 分支状态排列在相邻下方行。
- 回退边、取消边和重新激活边使用独立通道。
- 点击连线或标签后打开现有流转详情抽屉。

### 同状态操作

当 `from_state_id === to_state_id` 时，流转仍是完整业务流转，但不再生成 SVG 自环路径。

- 操作以紧凑按钮形式排列在所属节点下方。
- 按钮展示 `action_name`，并保持稳定的 `sort_order` 和 ID 排序。
- 点击按钮后打开同一个流转详情抽屉。
- 节点布局和画布边界需要包含操作按钮所占空间。
- 新增流转、保存流转、运行时按钮计算和高级配置协议保持不变。

## 自动布局

### 主图识别

- 初始状态所在的弱连通分量作为主图，而不是只使用从初始状态正向可达的节点。
- 这样仅向主图汇入的状态不会被错误识别为孤立状态。
- 自环不参与层级计算。

### 分层规则

- 初始状态固定在第一层。
- 正向主流程边推动目标状态向右分层。
- 次级零入度状态如果流向已分层目标，应放在目标前一层，而不是回到第一层。
- 同层节点按稳定状态顺序纵向排列。
- 行间距包含节点高度、同状态操作按钮区和安全间距。
- 不属于主图的其他启用弱连通分量依次放在主图下方。
- 停用状态统一放在所有启用区域下方，并采用独立间距。

默认需求模板整理后的预期结构：

- 第一列：待分派。
- 第二列：处理中、待确认。
- 第三列：已完成、已取消。
- 状态变化边连接这些节点；同状态操作排列在节点下方。

## 拖拽稳定性

### 拖拽开始

- 捕获当前完整边视图和画布边界快照。
- 记录当前视口偏移。
- 不立即把所有边切换为简化预览。

### 拖拽过程

- 只对起点或终点是当前拖拽节点的边生成轻量预览。
- 无关边继续使用拖拽开始时的完整路径快照。
- 同状态操作按钮随所属节点移动。
- 画布边界使用拖拽开始时的边界快照并结合当前节点位置，不因完整路由暂时缺失而收缩。
- 视口偏移保持不变。

### 拖拽结束

- 恢复完整整图路由。
- 不自动调用视口限制或适应视图，因此页面不跳动。
- 用户可显式点击“适应视图”重新居中。
- 坐标仅保存在前端状态中，仍需点击“保存流程图”才持久化。

## 模板状态复用与数据库清理

### 后续模板套用

模板状态按当前工作流定义内的稳定语义进行复用：

1. 优先匹配同名且启用的状态。
2. 没有启用匹配时，可匹配同名、同类别的禁用状态并重新启用。
3. 没有匹配时才创建新状态。
4. 模板未包含且仍被业务或历史记录引用的状态保持禁用。
5. 模板未包含且没有引用的状态可删除。

模板流转仍按模板完整重建，并将端点映射到复用后的状态 ID。

### 一次性数据清理

增加可重复执行的数据迁移，清理同一工作流定义内的禁用重复状态：

1. 查找禁用状态对应的同名启用状态。
2. 将需求、任务、Bug 的 `current_state_id` 迁移到启用状态。
3. 将操作历史的 `from_state_id` 和 `to_state_id` 迁移到启用状态。
4. 将条件配置中的状态引用迁移到启用状态（如存在）。
5. 确认没有剩余引用后物理删除禁用重复状态。
6. 没有唯一同名启用状态时跳过并记录，不做猜测性迁移。

迁移必须在事务内执行，任一引用更新或删除失败时整体回滚。

## 组件与模块边界

- `workflowAutoLayout.js`
  - 负责弱连通主图识别、稳定分层、操作区间距和停用状态区域坐标。
- `workflowEdgePath.js`
  - 只路由状态变化边。
  - 保留完整路由和拖拽相关边的轻量预览能力。
- 新的工作流画布投影纯函数模块
  - 将全部状态和流转拆分为启用状态、停用状态、状态变化边、同状态操作。
  - 生成拖拽时“关联边预览 + 无关边快照”的组合视图。
- `WorkflowDesigner.vue`
  - 渲染节点、状态变化边和节点操作按钮。
  - 管理抽屉选择、拖拽快照和视口冻结。
- 后端模板持久化服务
  - 复用语义相同的已有状态，避免每次套用都创建新状态。
- 数据迁移
  - 迁移引用并删除已有禁用重复状态。

## 错误处理

- 数据迁移发现零个或多个同名启用目标时，跳过该状态并输出明确记录。
- 拖拽快照中找不到某条边时，回退到该边的轻量预览，不隐藏边。
- 同状态操作的端点不存在或状态已被删除时，沿用现有无效端点处理，不渲染悬空按钮。
- 空图整理保持无错误提示行为。

## 测试与验收

### 自动测试

- 默认需求模板的 5 个启用状态形成三列两行结构。
- 仅向主图汇入的状态仍属于主图，不进入孤立区域。
- 停用状态位于独立区域，坐标稳定且不与启用节点重叠。
- 同状态操作不会进入边路由，但会生成稳定的节点操作列表。
- 所有状态变化边仍保留、确定且不修改业务字段。
- 拖拽时无关边路径保持字节级不变，关联边跟随节点。
- 拖拽开始和结束时视口偏移保持不变。
- 模板连续套用两次不增加状态数量，并复用状态 ID。
- 数据迁移后业务项和操作历史引用有效，重复禁用状态被删除。
- 无唯一迁移目标时不删除状态。

### 浏览器验收

- 使用当前 `cfg` 方案的需求、任务和 Bug 工作流。
- 检查重复禁用状态以虚线样式位于停用区域；执行数据迁移后不再出现已清理的重复状态。
- 点击“整理布局”后检查主流程、下方分支和回退通道。
- 检查每个同状态操作按钮可见并能打开对应流转抽屉。
- 拖拽每类节点，记录视口 transform 和无关边 path，确认过程与松手后页面不跳动。
- 不点击保存，确认验收过程不持久化坐标。

## 明确的设计取舍

- 画布保留所有业务按钮，但用不同视觉语法区分“状态变化”和“同状态操作”。
- 停用状态不隐藏，避免失去重新启用入口；通过样式和分区消除混淆。
- 拖拽结束后允许完整路由发生必要变化，但视口不自动移动。
- 数据清理优先保证引用完整性，不对缺少唯一映射的状态执行删除。
