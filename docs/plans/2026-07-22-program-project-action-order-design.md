# 项目集与项目状态动作排序设计

## 背景

项目集列表将编辑按钮排在启动、暂停、关闭、激活等状态动作之前。项目集树中的项目节点也存在相同的显示顺序。状态动作是推进生命周期的主要操作，应在维护性编辑操作之前呈现。

## 决策

仅调整前端 `ProgramsView.vue` 的按钮渲染顺序：

- 项目集节点：状态动作在编辑按钮前，新增项目集、新增项目和删除仍位于编辑之后。
- 项目节点：`WorkflowActionButtons` 在编辑按钮前，新增项目和删除仍位于编辑之后。
- `ProjectsView.vue` 已满足相同的项目操作顺序，不修改运行时代码。

## 不变项

- 启动、暂停、关闭、激活的显示条件、权限校验和接口调用不变。
- 编辑、新增和删除的权限与行为不变。
- 不新增 API、状态或数据迁移。

## 验收

- 项目集在规划中、暂停中、执行中、关闭时，状态动作均先于编辑显示。
- 项目集树中的项目节点，工作流状态动作先于编辑显示。
- 独立项目列表继续保持状态动作先于编辑显示。
