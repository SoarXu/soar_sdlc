# 工作流自动整理布局设计

## 背景

当前工作流模板保存并复制固定节点坐标。模板套用后，设计器只将现有节点居中到视口，不会根据流转关系重新排版。现有连线算法又为每条边独立选择起终点中线，平行边、反向边和跨层边容易复用同一通道及标签位置，导致流程图紧凑、交错且难以阅读。

## 目标

- 套用模板成功后自动整理节点与连线。
- 在设计器工具栏提供“整理布局”按钮，允许用户随时重新整理当前流程图。
- 主流程结构清晰，分支、回退、环路和平行流转可以辨认。
- 整理布局只更新前端节点坐标，不自动保存，不改变任何工作流业务配置。
- 整理完成后自动适应视图，完整展示整理结果。

## 非目标

- 不改变后端接口、数据库结构或工作流运行时语义。
- 不引入新的流程图框架或第三方自动布局依赖。
- 不在每次打开工作流时覆盖用户已经保存的人工布局。
- 不自动提交整理后的节点坐标。

## 方案选择

采用项目内纯函数实现“分层自动布局 + 连线通道分配”。相比仅放大模板固定间距，该方案同时适用于系统模板、已有方案和用户后续编辑的图；相比引入 Dagre 或 ELK，可以保持当前 Vue + SVG 设计器架构，避免增加依赖和二次适配成本。

## 布局设计

### 节点分层

1. 优先从定义的初始状态开始遍历；没有初始状态时，优先选择入度为零的节点，再按稳定顺序补充其余节点。
2. 根据前向流转计算节点层级，主流程从左到右排列。
3. 同层节点按稳定顺序纵向排列，避免每次点击整理产生无意义的位置跳动。
4. 回退边和环路不参与提高目标节点层级，避免循环导致层级无限增长。
5. 无法从主流程访问的孤立节点放在主图下方的独立区域。
6. 使用固定节点尺寸和最小间距：水平间距约 240px，垂直间距约 120px，并保留画布边距。

### 连线路由

1. 普通前向边优先从节点右侧连接到目标节点左侧。
2. 同一节点对之间的多条流转分配不同的平行通道和标签位置。
3. 跨层边分配独立轨道，避免穿过中间节点。
4. 回退边和自环边从主流程下方绕行，避免与前向主线重叠。
5. 路由结果保持正交折线和圆角，延续现有视觉风格。

## 交互流程

### 套用模板

1. 用户确认覆盖当前流程图。
2. 前端调用现有套用模板接口。
3. 前端加载返回的节点和流转。
4. 自动执行整理布局。
5. 自动适应视图并提示模板已套用。
6. 用户检查结果后，点击现有“保存流程图”按钮持久化坐标。

### 手动整理

1. 用户点击工具栏“整理布局”。
2. 页面提示该操作会重新排列全部节点。
3. 用户确认后，前端对当前图执行整理。
4. 自动适应视图。
5. 用户可继续拖动微调，或点击“保存流程图”持久化。

取消确认时保持当前图不变。空图点击整理时不报错，可给出轻量提示。

## 代码边界

- 新增独立的工作流布局纯函数模块，负责节点分层、稳定排序和坐标计算。
- 扩展连线路由纯函数，使其基于全部节点和边分配通道，而不是逐边孤立计算。
- `WorkflowDesigner.vue` 只负责触发整理、应用计算结果、确认交互和适应视图。
- 后端模板、图保存协议和数据库模型保持不变。

## 测试与验收

- 纯函数测试覆盖线性流程、分支、合并、环路、回退边、孤立节点和平行边。
- 验证同一输入多次整理得到相同坐标和路由。
- 验证整理不修改节点 ID、名称、类别、启用状态、流转端点或高级配置。
- 组件契约测试覆盖工具栏按钮、确认行为和模板套用后自动整理。
- 运行前端全量测试与生产构建。
- 在桌面视口实际套用需求、任务、Bug 模板，检查节点不重叠、连线可辨认、标签不遮挡节点，并验证整理结果只有在保存后才持久化。
