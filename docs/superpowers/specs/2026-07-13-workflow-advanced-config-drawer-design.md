# 工作流高级配置抽屉设计

## 1. 背景

当前工作流设计器将条件路由、动作表单、校验与按钮、通知自动化放在右侧属性栏底部。随着配置项增多，存在以下问题：

- 右侧栏宽度不足，路由表、表单字段和角色配置难以横向组织。
- 基础属性和高级配置共用一条长滚动区域，用户容易失去当前流转上下文。
- “校验与按钮”混合了门禁、处理人和展示设置，职责不清晰。
- 高级配置是否已经修改、是否已经应用、是否已经保存到后端，缺少明确反馈。

本设计只调整工作流设计器的前端信息架构和编辑交互，不改变数据库结构、工作流定义 API 或运行时执行协议。

## 2. 目标

- 保持流程画布稳定，打开高级配置时不移动或缩放节点。
- 为复杂配置提供足够的横向空间和清晰分组。
- 区分抽屉草稿、已应用到当前流转、已保存到后端三个状态。
- 防止关闭抽屉、切换工作流或离开页面时意外丢失修改。
- 完整保留现有受支持配置和未知历史配置。

## 3. 总体布局

### 3.1 基础属性栏

原右侧属性栏保留以下基础配置：

- 流转名称和动作键。
- 源状态和目标状态。
- 允许执行角色摘要，只读展示并通过高级配置编辑。
- 基础下一处理人规则摘要。
- 启用状态。
- 删除流转。

属性栏增加“高级配置”入口，并显示配置摘要，例如“已配置 3 项”。

### 3.2 高级配置抽屉

- 抽屉从页面右侧覆盖画布打开。
- 桌面环境下宽度为 `clamp(520px, 42vw, 640px)`，在常见笔记本和全高清显示器上自适应。
- 抽屉高度跟随设计器可用高度，标题和底部操作栏固定，只有内容区域滚动。
- 表单网格使用 `repeat(auto-fit, minmax(190px, 1fr))`；空间充足时显示两列，空间不足时自动变为单列。
- 抽屉不触发画布重排、缩放或节点坐标变化。
- 标题显示当前流转名称以及“源状态 -> 目标状态”。
- 标题区域显示草稿状态，例如“未应用修改”。
- 底部操作栏固定，内容区域独立滚动。

## 4. 信息架构

抽屉左侧使用固定导航，分为五个栏目。

### 4.1 流转规则

包含：

- 路由字段。
- 路由模式：自动路由、手动选择、自动且允许覆盖。
- 允许覆盖角色。
- 字段值到目标状态的映射。
- 终态门禁或其他受支持 validator。

Bug 默认类型路由显示为“Bug 类型字典自动路由”。界面不展开九条类型映射，也不允许同时添加静态类型路由，避免产生第二份业务规则。

### 4.2 处理人与权限

包含：

- 允许执行角色。
- 主要处理人来源及目标角色。
- 回退处理人来源及回退角色。
- 是否允许手动指定处理人。
- 需要时展示固定用户、项目角色等来源的附加字段。

界面需要明确区分：允许执行角色决定“谁能执行”，目标角色决定“执行后谁接手”。

### 4.3 动作表单

包含：

- 表单标题和提交文字。
- 字段键、显示名称、组件类型、必填状态。
- 下拉选项或受支持的数据字典。
- 字段增删和排序。

字段键必须稳定且唯一。条件路由引用的字段必须存在于动作表单，或者由受支持的数据字典配置提供。

### 4.4 按钮展示

包含：

- 按钮样式。
- 列表位置：主操作、更多操作、隐藏。
- 动作类别。
- 详情可见和列表可见。
- 执行前确认。

该栏目只负责交互呈现，不混入业务门禁或处理人规则。

### 4.5 通知

包含：

- 执行前通知开关、接收人和标题。
- 完成后通知开关、接收人和标题。
- 当前后端支持的通知接收人选项。

当前范围只包含系统站内通知，不增加邮件、短信或企业微信渠道。

## 5. 编辑和保存模型

### 5.1 草稿隔离

打开抽屉时，前端深拷贝当前流转的高级配置。抽屉内五个栏目共同编辑该草稿，不直接修改画布中的流转对象。

### 5.2 两级保存

1. 点击“应用配置”：校验抽屉草稿，并将其写回当前流转对象。
2. 点击页面顶部“保存工作流”：将整个工作流图提交到后端。

“应用配置”不会直接调用后端保存接口，避免编辑一个流转时意外提交其他尚未完成的工作流修改。

### 5.3 取消和退出

- 没有草稿修改时，取消、关闭按钮和 Esc 直接关闭抽屉。
- 存在未应用修改时，关闭抽屉必须确认是否放弃。
- 切换工作流、应用模板或离开页面时，也必须处理未应用草稿。
- “清空本页配置”只清空当前栏目，并计入草稿修改状态。

## 6. 状态反馈与校验

每个栏目显示以下状态之一：

- 未配置。
- 已配置。
- 存在错误。

已配置栏目显示蓝点，存在错误的栏目显示红点。点击“应用配置”时执行统一校验：

- 字段键不能为空、不能重复。
- 下拉字段必须配置有效选项或受支持字典。
- 静态路由必须有字段、字段值和有效目标状态。
- 字典路由不能与静态路由同时存在。
- 手动路由和覆盖模式必须满足相应角色配置。
- 项目角色处理人来源必须选择角色。
- 通知启用后必须选择接收人并填写标题。

校验失败时不应用配置，自动切换到第一个错误栏目，并在控件下方显示具体错误。

## 7. 历史配置保护

- 读取时完整保留当前序列化工具不认识的历史字段。
- 如果运行时组件类型不受支持，抽屉显示警告和只读原始配置。
- 未完成迁移前，不允许通过设计器覆盖保存该流转。
- 受支持配置在打开、切换栏目、应用和最终保存后不得丢失字段。

## 8. 错误处理

- 抽屉前端校验错误保留全部草稿内容。
- 工作流最终保存失败时保留页面状态和已应用配置，不关闭设计器。
- 后端返回的结构校验错误应映射到对应栏目；无法定位时显示抽屉级或页面级错误。
- 禁止使用静默删除、自动修复未知字段或自动替换无效角色的方式处理错误。

## 9. 实现边界

本次包含：

- 拆分工作流设计器高级配置 UI。
- 新增抽屉草稿、脏状态、退出确认和栏目状态计算。
- 复用并扩展现有工作流配置序列化工具。
- 为现有配置增加前端结构校验和错误定位。

本次不包含：

- 数据库迁移。
- 工作流定义 API 变更。
- 新增 validator、处理人来源或通知渠道。
- 移动端或移动视口适配验收。
- 修改工作流运行时业务语义。

## 10. 测试与验收

### 10.1 自动测试

- 打开抽屉后修改草稿不会立即改变当前流转。
- 应用后配置正确写回，取消后配置保持不变。
- 有未应用修改时关闭、切换或离开会触发确认。
- 五个栏目的已配置和错误状态计算正确。
- 条件、表单、处理人、按钮、通知配置序列化往返无数据丢失。
- Bug 类型字典路由不会产生静态类型映射。
- 未知历史配置保持只读且不能被覆盖。
- 前端全量测试和 Vite 生产构建通过。

### 10.2 桌面尺寸验收

分别使用以下三个独立固定桌面视口运行验收，每次运行期间不移动或动态调整视口：

- `1366x768`：常见小屏笔记本。
- `1440x900`：项目基准桌面尺寸。
- `1920x1080`：常见全高清显示器。

不增加移动端视口测试。每个桌面尺寸均验证：

- 抽屉覆盖画布，节点位置和缩放不变化。
- 标题、左侧导航、内容滚动和固定底栏不重叠。
- 长动作表单和多条条件路由可正常编辑。
- 错误栏目能够自动定位。
- 取消、Esc、关闭、应用配置和最终保存行为符合两级保存模型。
- 抽屉宽度处于 520px 到 640px 之间，表单根据可用空间在一列和两列之间自然切换。
