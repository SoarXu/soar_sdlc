# 工作流状态 ID 化收尾总结报告

## 1. 报告结论

截至 2026-07-17，本轮已确认的工作流状态 ID 化收尾范围全部完成，未发现已确认但未实现的功能。需求、任务、Bug、项目、迭代五类业务对象均以工作流定义 ID 和状态节点 ID 运行；状态名称只承担展示语义，状态分类承担阶段判断，业务关联和流转不再依赖状态字符串。

本报告结论来自隔离分支代码、隔离 MySQL 数据库的实际 schema、迁移往返、全量测试和生产构建。原 PRD 第 9 章尚待产品确认的事项不属于本轮已确认范围，仍按待决策项管理。

## 3. 数据库最终状态

### 3.1 迁移链

- `20260717_001_project_iteration_state_identity.py`：为项目、迭代增加可空 ID 字段，按既有系统定义和旧状态完成确定性回填，并建立索引和外键。
- `20260717_002_remove_workflow_state_strings.py`：迁移前审计五类对象、定义初始状态、流转端点和启用的旧处理人规则；审计通过后收紧非空约束，删除 5 个旧列和旧处理人规则表。
- 最终版本：`20260717_002 (head)`。

### 3.2 约束与索引

运行模型共有 15 个工作流身份外键：

- 需求、任务、Bug、项目、迭代各 2 个：`workflow_definition_id` 和 `current_state_id`，共 10 个。
- `workflow_definitions.initial_state_id` 1 个。
- `workflow_states.definition_id` 1 个。
- `workflow_transitions.definition_id/from_state_id/to_state_id` 3 个。

上述字段均有单列或联合查询索引。状态操作历史 `status_operation_log` 另保留 3 个定义/来源状态/目标状态外键，用于历史关联和名称快照，不计入 15 个运行模型外键。

外键保证记录指向存在的定义和状态；“当前状态必须属于该对象的定义”是跨行归属规则，MySQL 单列外键本身不能完整表达，因此由服务校验、迁移阻断审计和持续审计共同保证。隔离库本轮实查五类对象跨定义记录为 0。

### 3.3 处理人规则往返

`handler_transition_rules` 中原有启用规则只对应 `assignee_rule_config` 范围。破坏性迁移先确定性匹配到唯一流转，再写入该流转的 `handler_rule`；无法匹配、匹配不唯一或与已有 JSON 冲突时迁移会失败并报告记录 ID，不会猜测或静默丢弃。

本轮在隔离库执行 `20260717_002 -> 20260717_001 -> 20260717_002`：

| 检查项 | 往返前 | 降级至 001 | 再升级至 002 |
|---|---:|---:|---:|
| `assignee_rule_config` 处理人规则数 | 102 | 旧表重建 102 | 102 |
| 规则 JSON 总字节数 | 14121 | 不适用 | 14121 |
| 规则内容 SHA-256 | `911f7e...3e00` | 不适用 | `911f7e...3e00` |

完整指纹为 `911f7e611da9a4c697629dcb215ca21c22ca313f648554d417e066ff3caa3e00`。数量、总字节数和按流转 ID 排序后的内容指纹完全一致，可确认这 102 条规则在本次往返中未丢失或改写。

## 4. 运行流程核对

1. 创建五类业务对象时，系统解析对应工作流定义并写入定义的 `initial_state_id`，负责人是否存在不改变初始状态。
2. 查询时通过 `current_state_id` 关联状态节点，返回中文 `status_name` 和 `state_category`。
3. 动作发现按 `workflow_definition_id + current_state_id` 查找启用流转，再校验权限、条件和门禁。
4. 执行动作时根据 `action_key` 运行对应领域逻辑，根据 `to_state_id` 或 ID 条件路由决定目标节点。
5. 下一处理人只从选中流转的 `handler_rule` 解析；随后更新 `current_state_id` 和处理人。
6. 操作历史记录定义、来源状态、目标状态 ID，并保留状态中文名称快照。

直接启动、暂停、恢复、关闭、激活等项目端点均委托正式工作流流转；迭代自动启动也按状态 ID 执行。页面按钮依据后端返回的可执行动作展示，不再通过状态字符串推测动作。

需求、任务页面对项目终态的门禁统一读取 `project.state_category == "terminal"`。`20260717_001` 新增的项目激活流转带有专属迁移标记，降级时只删除该迁移实际插入的流转；升级前已存在且没有迁移标记的合法激活流转会被保留。

## 6. 未实现项与范围边界

本轮已确认的状态 ID 化收尾范围没有已知未实现项。

以下内容仍未实现，但属于原 PRD 第 9 章待产品确认项，不是本轮缺陷：已启用方案编辑限制、方案删除、停用后恢复、终态新增规则、节点处理模式，以及工作台展示/超时计算的节点级独立配置。未实现原因是这些行为会改变运行中业务数据和流程治理边界，现有 PRD 没有给出可执行规则，不能由技术实现代替产品决策。

Program 状态 ID 化、系统模板版本管理、新状态编码均为本轮明确不做，不计为未实现功能。
