# 工作流节点拖拽点击抑制设计

## 问题

节点使用 `mousedown` 开始拖拽、全局 `mouseup` 结束拖拽，同时节点本身绑定 `click` 打开配置抽屉。浏览器在拖拽松手后仍会派发 `click`，而此时拖拽状态已经清空，导致抽屉被误打开。

## 方案

采用移动阈值区分单击和拖拽：鼠标相对按下位置移动超过 4px 后，将本轮操作标记为拖拽。拖拽结束时记录需要抑制点击的节点 ID，并在紧随其后的节点点击处理中消费该标记；若浏览器没有派发点击，则在当前事件序列结束后自动清理标记。

正常原地单击仍打开抽屉。角标点击、连线选择、路径编辑和键盘操作保持现有行为。

## 测试

组件契约测试覆盖以下行为：节点点击通过独立处理函数进入；拖拽移动超过阈值后设置移动标记；拖拽结束时只为发生移动的节点设置点击抑制；节点点击命中抑制标记时不会调用 `selectState`，正常点击仍会调用。
