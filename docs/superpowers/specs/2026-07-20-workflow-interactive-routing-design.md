# 工作流画布交互式布线设计

## 背景

当前工作流画布有两个相互关联的可读性问题：

1. 同状态流转以独立的“操作 N”矩形常驻在节点下方。虽然已经从多个按钮收敛为单个入口，但该入口仍会与状态流转标签和连线路径争夺空间。
2. 拖拽节点期间，画布只用轻量预览路径替换与被拖节点相连的边，其他边保持拖拽开始时的旧路径；松手后再切回全量自动路由。因此拖拽过程中的画面不是最终结果，松手时会发生二次跳变。

用户确认采用以下方向：

- 同状态操作入口改为节点右上角数量角标。
- 自动布线作为默认模式。
- 用户可手工覆盖连接点和折线路径。
- 折线路径不显示常驻控制点，直接拖动水平或垂直线段进行调整。

## 目标

- 流程图默认只突出状态节点、跨状态流转线和流转标签。
- 拖拽节点期间显示的路径与松手后的路径完全一致。
- 自动布线无法满足阅读需求时，用户可以调整连接位置和正交折线路径。
- 手工结果保存后稳定回读，不被普通节点拖拽或页面重载静默覆盖。
- 画布几何配置与运行时按钮配置分离。

## 非目标

- 不实现任意角度或曲线连线，所有流转线继续保持水平、垂直的正交折线。
- 不实现完整的撤销/重做历史；仅支持按 Escape 取消当前正在进行的拖动。
- 不让手工路径自动避开后续新增的节点。手工模式以用户明确保存的路径为准。
- 不改变工作流运行时、权限、条件、处理人和动作表单语义。

## 同状态操作入口

### 节点角标

存在同状态流转的节点在右上角显示一个紧凑的数量角标。角标属于节点自身的交互区域，不再创建节点下方的独立矩形，也不增加自动布局行高、画布边界或边路由障碍物。

角标内容只显示同状态操作数量。鼠标悬停时显示“`N` 个同状态操作”提示；键盘和辅助技术使用包含状态名称及数量的 `aria-label`。没有同状态流转的节点不显示角标。

### 操作菜单

点击角标后继续使用现有 HTML 浮层菜单：

- 菜单按现有稳定顺序列出全部同状态流转。
- 点击菜单项打开现有流转详情抽屉。
- 停用流转保持灰色，但仍可进入抽屉重新启用。
- 点击空白区域、选择其他节点或流转、拖拽节点、平移画布、切换流程图或按 Escape 时关闭菜单。
- 菜单及角标均不参与边路由与画布尺寸计算。

## 实时节点拖拽

### 单一最终路由算法

拖拽节点时不再混合“冻结的完整路径”和“轻量预览路径”。每个动画帧最多执行一次与松手后相同的完整边投影：

1. 更新被拖节点坐标。
2. 在 `requestAnimationFrame` 中按当前全部节点、全部流转和全部手工配置生成完整边视图。
3. 渲染该完整边视图。
4. 松手时仅结束拖拽状态，不切换路由算法，不再额外重算另一套路径。

自动边在拖拽过程中持续避让全部节点和标签。手工边根据已保存的几何规则更新。现有拖拽期间的画布边界冻结与视口抑制逻辑保留，避免画布尺寸和视口偏移随路径变化而跳动。

### 性能边界

路由更新按动画帧合并，连续鼠标事件不会重复执行同一帧的路由计算。若一帧内收到多次坐标变化，仅使用最后一次坐标。测试以当前模板规模及明显高于当前规模的合成图验证拖拽可用性，但本次不引入 Web Worker。

## 手工连接点与路径

### 进入手工模式

流转默认使用自动路由。用户对连接端点或内部线段完成一次拖动后，该流转切换为手工模式。选中手工流转时使用现有选中高亮，不增加常驻折线点。

### 连接端点

选中连线后，起点和终点显示小型空心端点。用户可将端点拖到源节点或目标节点的任意一边。端点沿边连续吸附，并保留边缘安全间距，不限定为四边中点。

端点以边和边内比例保存，例如：

```json
{
  "side": "bottom",
  "ratio": 0.35
}
```

`ratio` 限制在 `0` 到 `1` 之间；实际渲染时再根据节点当前尺寸换算坐标，并将最终坐标限制在距离圆角至少 8px 的可连接区间内。因此节点移动后连接点仍保持在对应边的相对位置，也不会落在节点圆角上。

### 直接拖动线段

手工路径不显示折线控制点。每条内部水平或垂直线段增加不可见的宽命中区域：

- 水平线段显示纵向调整光标，只能改变 `y`。
- 垂直线段显示横向调整光标，只能改变 `x`。
- 拖动线段时同步更新它两端的中间路径点，始终保持正交。
- 与节点直接相连的首尾短线段不作为线段拖动目标；连接位置通过端点拖动调整。
- 重合点、零长度线段和多余共线点在每次更新后归一化删除。

按 Escape 时恢复本次拖动开始前的路径；松手后当前画面即为最终结果，不执行额外自动整理。

### 节点继续移动

手工边连接的节点被移动时：

- 起终点根据保存的边和比例实时更新。
- 首尾相邻线段跟随新端点伸缩。
- 中间路径点保持用户保存的位置。
- 路径继续保持正交，但不自动改变中间轨道。

自动边仍使用完整自动路由。这样同一张图可以同时存在自动边和手工边。

### 标签

流转标签继续由现有标签算法放置在手工折线的可用长线段上。用户调整路径后，标签随当前路径实时重新选择位置，但不改变手工路径本身。

## 数据模型

### 独立字段

`workflow_transitions` 新增可空 JSON 字段 `diagram_config`，并同步更新 ORM、Pydantic 保存/读取模型和图保存服务。不得将画布几何写入 `ui_config`，因为 `ui_config` 只允许运行时按钮相关字段，并由后端白名单校验。

自动模式使用 `NULL` 或空配置。手工模式的版本化结构为：

```json
{
  "version": 1,
  "routing_mode": "manual",
  "source_anchor": { "side": "bottom", "ratio": 0.35 },
  "target_anchor": { "side": "top", "ratio": 0.5 },
  "waypoints": [
    { "x": 356, "y": 418 },
    { "x": 612, "y": 418 }
  ]
}
```

`waypoints` 只保存端点之间的中间正交路径点，最多 32 个。服务端验证版本、模式、边名称、比例范围、有限数值、路径点数量以及相邻线段正交性。无效配置返回 422，不静默修正数据库内容。

### 兼容性

数据库迁移只新增可空字段，现有数据无需回填并继续使用自动路由。默认模板不写入手工路径。套用模板会按当前行为重新组织节点，并清除目标流程中原有手工路径。

## 抽屉与整理布局

流转详情抽屉增加“画布布线”状态：

- 自动布线：无手工配置。
- 手工布线：已保存端点或路径。
- 手工模式下提供“恢复自动布线”命令，清空 `diagram_config` 并立即重新生成自动路径。

“整理布局”确认文案明确说明会重新排列节点并清除全部手工路径。用户确认后先清空所有流转的 `diagram_config`，再执行自动布局和完整自动路由。取消确认时不改变节点或路径。

## 状态流与错误处理

- 打开其他节点、流转或操作菜单时，结束当前选择并关闭临时菜单。
- 正在拖动端点或线段时不允许同时平移画布或拖动节点。
- 当前流转被删除、图被重新加载或对象类型切换时，清理所有临时编辑状态。
- 保存失败时保留前端当前路径，显示现有错误反馈，允许用户修正或重试。
- 后端回读图后以服务器返回的 `diagram_config` 为准。

## 代码边界

- `WorkflowDesigner.vue`：交互状态、SVG 命中区域、端点/线段拖动、菜单角标和生命周期协调。
- `workflowEdgePath.js`：自动路径和手工路径统一投影为同一种边视图；复用现有标签及边界计算。
- 新增独立纯函数模块：解析、验证、归一化和更新手工正交路径，避免继续扩大组件职责。
- 后端工作流定义模型、视图和服务：持久化并校验 `diagram_config`。
- Alembic：新增可空 JSON 列。

## 测试策略

### 前端纯函数

- 自动边与手工边混合投影。
- 边内比例到实际端点坐标的换算。
- 水平和垂直线段拖动只改变允许轴。
- 删除零长度和多余共线点。
- 移动源节点或目标节点后，中间手工路径保持稳定。
- 手工路径标签与边界计算。

### 前端交互契约

- 节点角标替代节点下方“操作 N”矩形。
- 角标不增加布局行高、画布边界或路由障碍物。
- 拖拽节点期间与松手后使用同一完整边视图。
- 端点和线段拖动互斥于节点拖动及画布平移。
- Escape 回滚当前编辑。
- 恢复自动布线和整理布局清除手工配置。

### 后端

- 新字段保存与回读。
- 现有空配置兼容。
- 无效边名称、比例、非有限数值、斜线段和过多路径点返回 422。
- 模板套用及整理布局后的手工配置清理行为。

### 浏览器验收

- 当前需求流程图中不再出现节点下方操作矩形。
- 角标、菜单、长操作名称和停用操作均可正常使用。
- 慢速及快速拖动节点时，松手前后线路不跳变。
- 可以连续拖动端点和内部线段，并在保存、刷新后保持一致。
- 手工边和自动边同时存在时均可选择、打开抽屉和正确放置标签。
- 宽屏与窄屏下无文本裁切、菜单越界或无关视口位移。

## 验收标准

1. 常态画布不再渲染节点下方“操作 N”矩形。
2. 拖拽节点的最后一帧与松手后的首帧路径几何相同。
3. 用户可沿节点边连续调整连接位置，并可直接拖动内部正交线段。
4. 手工路径不显示常驻折线控制点。
5. 手工路径保存、刷新和再次编辑后保持一致。
6. 恢复自动布线及整理布局行为明确且可预测。
7. 现有无 `diagram_config` 数据无需迁移回填，运行时业务行为不变。
