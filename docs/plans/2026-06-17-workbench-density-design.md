# 工作台高密度视图设计

## 背景

工作台已经支持个人/全部视角、折叠迭代、泳道滚动和渐进展示。但当项目、迭代和需求继续增长时，卡片看板仍然信息密度偏低，用户需要频繁滚动和展开，管理效率会下降。

## 目标

1. 默认视图承载大量数据，避免卡片撑满页面。
2. 看板只承担排期和拖拽，不再作为大量数据的主浏览方式。
3. 单个工作项的详细信息和生命周期操作集中在右侧抽屉中完成。
4. 保留现有工作台数据接口和后端项目范围校验。

## 设计

工作台拆成三种视图：

- `列表`：默认视图。以表格展示工作项，一行一个工作项，支持分页、筛选和快速操作入口。适合大量需求、任务、用例和 Bug 的日常处理。
- `看板`：排期视图。保留按迭代和类型组织的拖拽能力，但卡片改为迷你条目，只展示类型、标题、负责人、状态和项目，点击后打开右侧抽屉。
- `统计`：概览视图。展示按迭代聚合的需求、任务、用例和 Bug 数量，用于快速判断当前阶段负载。

右侧抽屉展示选中工作项的完整信息和生命周期按钮，避免在列表/看板中堆叠过多按钮。

## 交互规则

1. 默认进入 `列表` 视图。
2. `我的工作` / `全部工作`、迭代、类型、负责人、关键词筛选对三种视图都生效。
3. 列表分页可设置每页条数。
4. 看板中仍支持拖拽到目标迭代，后端继续校验工作项是否属于目标迭代项目范围。
5. 点击列表行标题或看板迷你条目打开右侧抽屉。
6. 抽屉内可以进入详情页，也可以直接执行当前允许的生命周期操作。

## 验收标准

- 大量工作项默认以表格承载，不再使用大卡片作为主视图。
- 看板条目高度显著降低，单屏可显示更多条目。
- 工作项操作入口不丢失。
- 拖拽移动仍能调用原后端接口并保留失败回滚。
- 构建和后端测试通过。
