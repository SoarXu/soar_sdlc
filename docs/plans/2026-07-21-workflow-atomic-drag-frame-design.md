# 工作流原子拖拽帧设计

## 问题

节点拖拽期间可能出现节点已经移动、连接点和连线路径仍停留在旧位置的中间画面。全量布线计算的耗时会放大该现象，但根本问题是节点坐标与连线结果没有作为同一个渲染快照发布。

## 目标

拖拽期间每个显示出来的画面必须满足：节点位置、连接点、自动或手工连线路径均基于同一组节点坐标。系统负载较高时允许跳过中间鼠标位置，但不允许展示节点和连线不一致的帧。松手时必须处理最新鼠标位置，并保持松手前后的路径一致。

## 设计

新增拖拽帧构建器。它接收当前状态列表、需要绘制的流转、拖拽节点 ID 和最新目标坐标，创建不修改业务状态的节点快照，并使用现有完整布线算法计算该快照对应的全部连线视图。返回值将节点快照和连线视图封装在同一个对象中。

设计器不再在动画帧中直接修改业务节点坐标。每个 `requestAnimationFrame` 先完整构建拖拽帧，计算结束后一次性替换帧引用。模板中的节点与连线均从该引用读取，因此浏览器只能看到上一帧或完整的新帧。

鼠标松开时先刷新最后一个待处理坐标，再把拖拽帧中的最终节点坐标提交到业务状态，随后清除预览帧。完整布线算法是确定性的，因此提交前后的路径保持一致。

## 测试

- 实际调用自动和手工布线，验证移动节点后的节点边界与所有关联边端点一致。
- 验证帧构建过程不修改输入状态。
- 验证组件节点和连线都读取同一拖拽帧。
- 验证拖拽期间不直接修改业务节点，松手时才提交最后坐标。
- 运行前端全量测试与生产构建。
