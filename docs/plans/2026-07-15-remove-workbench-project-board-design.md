# 工作台项目看板移除设计

## 背景

工作台中的“项目看板”与项目、迭代及工作项列表能力重复，实际使用价值有限，同时增加了工作台接口的数据聚合开销和前端维护成本。本次完整移除该视图，不保留隐藏入口或兼容响应字段。

## 范围

- 删除工作台“项目看板”页签。
- 删除顶部“看板工作项”统计。
- 删除项目看板的分组、筛选、卡片展示、工作流动作加载和专属样式。
- 删除后端 `project_board` 响应模型、数据聚合和分组维度。
- 删除或调整前后端项目看板相关测试。
- 保留待处理、未分派、异常中心、我发起/关注等工作台能力。
- 保留同期加入的异常中心筛选、测试单、Bug 类型和动态工作流等无关改动。

## 实现方式

基于当前工作区逐项删除项目看板专属代码，不整体回退历史提交。前端不再读取 `project_board`，后端 `GET /dashboard/workbench` 不再返回该字段，也不再执行项目看板数据查询和组装。

## 数据流

删除后，工作台接口只组装个人待处理、未分派、跟踪项和异常中心数据。前端视图模型只规范化这些队列并生成对应入口和统计，工作流批量动作查询也只覆盖仍保留的队列。

## 兼容性

`project_board` 从工作台响应中直接删除，属于内部前后端同步变更。仓库内没有发现工作台页面之外的消费者，因此不保留废弃字段。

## 验证

- 前端视图模型测试确认入口和统计中不再出现项目看板。
- 前端构建及相关单元测试通过。
- 后端工作台 API 测试确认响应中不再包含 `project_board`。
- 后端工作台相关测试通过。
- 全仓搜索确认运行时代码不再引用 `project_board`、`buildProjectBoard` 或项目看板专属类型。
